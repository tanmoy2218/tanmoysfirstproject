Introduction to HTML Media

First Principle: The Web is More Than Text and Images

The fundamental truth is that a webpage should be able to deliver any kind of content, not just static text and pictures. For years, this was a major problem. To play a video or audio file, browsers had to rely on third-party plugins like Adobe Flash, QuickTime, or Silverlight. This was inefficient, insecure, and inconsistent across different computers.

The Core Problem

How can we embed video and audio into a webpage in a standardized, native, and plugin-free way? We need a universal system that works on every modern browser, from a desktop PC to a mobile phone, without requiring the user to install anything extra.

The Logical Solution

The solution was to create dedicated HTML tags whose sole purpose is to embed and control media: the <video> and <audio> tags introduced in HTML5.

The Video Element

Basic Video Implementation

The most basic implementation uses the src attribute:

<video src="my-awesome-video.mp4"></video>

However, this only shows the first frame of the video as a static image without any controls.

The Controls Attribute

The controls attribute is a boolean attribute that adds browser-native controls to the video player.

<video src="my-awesome-video.mp4" controls></video>

This creates a fully functional video player with play/pause, timeline, volume controls, and fullscreen option.

Video Attributes

Several attributes control the behavior and appearance of video elements:

width and height

Sets the dimensions of the player, just like with an <img> tag.

<video src="video.mp4" width="640" height="360" controls></video>

autoplay

Makes the video start playing as soon as the page loads.

Important: Modern browsers block autoplay with sound. To make autoplay work, you must also add the muted attribute.

<video src="video.mp4" autoplay muted controls></video>

loop

Makes the video automatically restart from the beginning when it finishes.

<video src="video.mp4" loop controls></video>

poster

Specifies an image to display before the video is played, like a YouTube thumbnail.

<video src="video.mp4" poster="images/video-thumbnail.jpg" controls></video>

Browser Compatibility & Multiple Sources

Not all browsers support the same video formats. Chrome might prefer .webm, while Safari might only support .mp4.

The solution is to provide multiple formats using the <source> element inside the <video> tag.

Robust & Professional Method

<video controls poster="images/thumbnail.jpg">
  <source src="videos/my-video.webm" type="video/webm">
  <source src="videos/my-video.mp4" type="video/mp4">
  Sorry, your browser doesn't support embedded videos.
</video>

The browser will try each source in order and use the first format it supports. Common video formats include:

  • .mp4 (H.264 codec): Most widely supported format
  • .webm (VP8/VP9 codec): Open-source format supported by Chrome and Firefox
  • .ogg (Theora codec): Older open-source alternative

Simple & Direct Method

<video src="movie.mp4" controls></video>

This method is simpler but risks incompatibility with browsers that don't support your chosen format.

The Audio Element

Basic Audio Implementation

The <audio> tag works almost identically to the <video> tag, just without the visual component.

<audio src="audio/theme-song.mp3" controls></audio>

This creates an audio player with play/pause, timeline, and volume controls.

Audio Attributes

The <audio> tag supports the same attributes as <video>:

<audio src="audio/theme-song.mp3" autoplay loop controls></audio>

Multiple Audio Sources

Just like with video, you can provide multiple audio formats for better browser compatibility.

<audio controls>
  <source src="audio/sound.ogg" type="audio/ogg">
  <source src="audio/sound.mp3" type="audio/mpeg">
  Your browser does not support the audio element.
</audio>

Advanced Media Features

Accessibility: The <track> Element

The <track> element allows you to add timed text tracks, such as subtitles or captions, making your media accessible to more users.

Adding Subtitles and Captions

<video controls>
  <source src="video.webm" type="video/webm">
  <source src="video.mp4" type="video/mp4">
  
  <!-- English Captions for the hearing impaired -->
  <track src="captions_en.vtt" kind="captions" srclang="en" label="English">

  <!-- Spanish Subtitles for translation -->
  <track src="subtitles_es.vtt" kind="subtitles" srclang="es" label="EspaÃ±ol">
</video>

Track Attributes

  • src: Path to the WebVTT (.vtt) file
  • kind: Type of track (captions, subtitles, descriptions)
  • srclang: Language of the track (e.g., "en" for English)
  • label: Name that appears in the player's captions menu

Track Kinds

  • captions: Transcription of dialogue and important sounds for users who can't hear audio
  • subtitles: Translation of dialogue into another language
  • descriptions: Separate audio description for visually impaired users
