Introduction to Forms

The First Principle: The Web Needs to Be a Two-Way Street

The fundamental truth is that a website isn't just a brochure for you to read. For the web to be useful, it needs a way to collect information from the user and send it back to the server.

Without forms, you couldn't log in, search for a video, buy a product, post a comment, or send a message. The web would be a read-only library.

The Core Problem

How do we create a standardized, reliable system to:

  • Display interactive fields for a user to fill in (text boxes, checkboxes, dropdowns)
  • Package the user's data neatly
  • Send that package to a specific destination on a server
  • Tell the server how the data is being sent

The logical solution to this entire problem is the HTML <form> element.

Basic Form Structure

The <form> Element

The <form> tag acts as the main container that tells the browser, "Everything inside here is part of one single submission."

<form>
  <!-- form elements go here -->
</form>

The <input> Element

This is the most common form tag. It's a self-closing tag that creates an input field. Its behavior changes based on its type attribute.

<input type="text">

The Need for Meaning - The <label>

Input fields are meaningless without a description. The <label> tag provides a descriptive text for form fields.

<label>First Name:</label>
<input type="text">

The Need for Connection - id and for

To create a direct link between a label and its input, we use the id and for attributes.

<label for="firstName">First Name:</label>
<input type="text" id="firstName">

This connection improves usability (clicking the label focuses the input) and accessibility (screen readers can announce the relationship).

The Need for Data Identification - The name Attribute

The name attribute serves as the "data label" or "key" for the submitted value. It's what the server uses to identify the data.

<label for="firstName">First Name:</label>
<input type="text" id="firstName" name="firstName">

When submitted, the browser creates a package like: firstName=(whatever the user typed)

The Need for Submission - The Submit Button

We need a way for users to submit the form data. The simplest way is with <input type="submit">.

<form>
  <label for="firstName">First Name:</label>
  <input type="text" id="firstName" name="firstName">
  
  <br><br>
  
  <label for="lastName">Last Name:</label>
  <input type="text" id="lastName" name="lastName">
  
  <br><br>
  
  <button type="submit">Submit</button>
</form>

A Better Button - The <button> Element

The <button> element is more flexible than <input type="submit"> because it can contain other HTML elements.

<!-- OLD WAY -->
<input type="submit" value="Submit Your Order">

<!-- NEW, BETTER WAY -->
<button type="submit">
  <strong>Submit</strong> Your Order
</button>

The type="submit" attribute tells the button to act as a form submit button.

Input Types

Text Input

The most basic input type for single-line text.

<label for="username">Username:</label>
<input type="text" id="username" name="username">

Password Input

Masks the input with dots or asterisks for sensitive information.

<label for="password">Password:</label>
<input type="password" id="password" name="password">

Email Input

Designed for email addresses with built-in validation.

<label for="email">Email:</label>
<input type="email" id="email" name="email">

Number Input

For numerical input with optional min and max attributes for validation.

<label for="age">Age (18-99):</label>
<input type="number" id="age" name="age" min="18" max="99">

Date Input

Displays a user-friendly calendar date picker.

<label for="birthdate">Date of Birth:</label>
<input type="date" id="birthdate" name="birthdate">

Selection Inputs

Radio Buttons - "Select One"

Radio buttons allow users to select exactly one option from a predefined list.

All radio buttons in a single group must share the same name attribute.

<label>T-Shirt Size:</label>
<br>
<input type="radio" id="sizeS" name="shirtSize" value="small">
<label for="sizeS">Small</label>
<br>

<input type="radio" id="sizeM" name="shirtSize" value="medium">
<label for="sizeM">Medium</label>
<br>

<input type="radio" id="sizeL" name="shirtSize" value="large">
<label for="sizeL">Large</label>

Output




The value attribute specifies what data will be sent to the server if that option is selected.

Checkboxes - "Select Many"

Checkboxes allow users to select multiple options from a predefined list.

<label>Pizza Toppings:</label>
<br>
<input type="checkbox" id="toppingPep" name="toppings" value="pepperoni">
<label for="toppingPep">Pepperoni</label>
<br>

<input type="checkbox" id="toppingMush" name="toppings" value="mushrooms">
<label for="toppingMush">Mushrooms</label>
<br>

<input type="checkbox" id="toppingOni" name="toppings" value="onions">
<label for="toppingOni">Onions</label>

Output




Checkboxes that are part of the same question should share the same name attribute.

Dropdown Menus - The <select> Element

Dropdown menus are ideal when you need users to select one option from a long list.

<label for="country">Country:</label>
<br>
<select id="country" name="country">
  <option value="">--Please choose an option--</option>
  <option value="in">India</option>
  <option value="us">USA</option>
  <option value="uk">United Kingdom</option>
  <option value="au">Australia</option>
</select>

Output


The <select> tag is the container, and each <option> represents a choice in the dropdown.

Advanced Inputs

The Need for Long-Form Text - <textarea>

The <textarea> element is designed for multi-line text input like comments or addresses.

<label for="comments">Any special instructions?</label>
<br>
<textarea id="comments" name="comments" rows="4" cols="50"></textarea>

The rows and cols attributes control the visible height and width of the text area.

File Input

Allows users to select files from their device to upload.

<label for="avatar">Upload your avatar:</label>
<input type="file" id="avatar" name="avatar" accept="image/*">

The accept attribute can limit the types of files users can select.

Range Input

Creates a slider control for selecting a value from a range.

<label for="volume">Volume:</label>
<input type="range" id="volume" name="volume" min="0" max="100" step="5">

The min, max, and step attributes define the range and increment values.

Color Input

Provides a color picker interface for selecting colors.

<label for="bgColor">Background Color:</label>
<input type="color" id="bgColor" name="bgColor" value="#ffffff">

Form Attributes

Action Attribute

Specifies where to send the form data when the form is submitted.

<form action="/submit-form" method="POST">
  <!-- form elements -->
</form>

Method Attribute

Defines how to send the form data (GET or POST).

  • GET: Appends form data to the URL (visible to users)
  • POST: Sends form data in the HTTP request body (not visible in URL)
<form action="/submit-form" method="POST">
  <!-- form elements -->
</form>

Placeholder Attribute

Provides a hint or example of what to enter in an input field.

<label for="email">Email:</label>
<input type="email" id="email" name="email" placeholder="user@example.com">

Required Attribute

Specifies that an input field must be filled out before submitting the form.

<label for="username">Username:</label>
<input type="text" id="username" name="username" required>

Pattern Attribute

Defines a regular expression that the input's value must match.

<label for="zipcode">Zip Code:</label>
<input type="text" id="zipcode" name="zipcode" pattern="[0-9]{5}" title="Five digit zip code">
